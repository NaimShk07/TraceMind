# UI Guidelines

TraceMind should feel like a premium developer tool.

Think:

Linear

GitHub

Raycast

Vercel

Not a typical dashboard template.

---

# Design Philosophy

Minimal.

Fast.

Professional.

Developer-first.

Every screen should communicate confidence.

---

# Theme

Dark mode first.

Light mode optional.

Use modern neutral colors.

Avoid bright gradients everywhere.

Use accent color sparingly.

---

# Layout

Three-column layout.

Sidebar

↓

Content

↓

Inspector Panel

Keep generous spacing.

---

# Sidebar

Contains

Dashboard

Repositories

Timeline

Investigations

Settings

Collapsed by default on mobile.

---

# Typography

Use modern sans-serif font.

Hierarchy

Page title

↓

Section title

↓

Card title

↓

Body text

↓

Metadata

Avoid excessive font weights.

---

# Cards

Rounded corners.

Soft shadows.

Subtle borders.

Comfortable padding.

Hover interaction.

---

# Buttons

Primary

Secondary

Ghost

Danger

Consistent sizes.

---

# Tables

Clean.

Dense.

Developer-friendly.

Support sorting.

Support search.

Support keyboard navigation.

---

# Timeline

Commits displayed as timeline cards.

Show:

Author

Message

Date

Hash

Files changed

Click to inspect.

---

# AI Chat

Looks similar to ChatGPT.

Streaming responses.

Code blocks.

Syntax highlighting.

Suggested prompts.

No unnecessary decorations.

---

# Evidence Panel

Every AI answer should include:

Confidence

Relevant commits

Files

Diff preview

Reasoning

Allow expanding sections.

---

# Loading States

Skeleton loaders.

No spinners when avoidable.

Streaming preferred.

---

# Empty States

Helpful illustrations.

Explain next action.

Example:

Import a repository to begin investigation.

---

# Icons

Use Lucide Icons.

Consistent sizing.

Avoid emoji.

---

# Animations

Fast.

150–250ms.

Fade.

Slide.

Scale.

Avoid slow animations.

---

# Responsiveness

Desktop first.

Tablet supported.

Mobile usable.

Sidebar collapses automatically.

---

# Accessibility

Keyboard navigation.

Visible focus states.

Sufficient contrast.

Semantic HTML.

ARIA labels when needed.

---

# UX Principles

Reduce clicks.

Surface important information first.

Always show system status.

Never leave users wondering what is happening.

Every screen should answer:

What happened?

What can I do next?

---

# Product Personality

Professional.

Intelligent.

Calm.

Never playful.

Never gimmicky.

Designed for engineers.
